# Requirements – TicTacToe Web Application

## 1. Personas

### Persona 1: Office Worker (Parent / Boss)

The office worker is a professional who mainly uses a desktop computer during work breaks. He prefers simple, fast, and reliable web applications that require no installation or setup. He plays TicTacToe with his child during lunch breaks as a short relaxing activity.

He values usability, clarity, and responsiveness. The application must work directly in the browser and support quick interactions without technical complexity.

---

### Persona 2: Student (Child)

The student uses a smartphone and is highly familiar with mobile applications. They prefer intuitive touch-based interfaces and fast response times. The student enjoys casual gaming and playing against their parent during school breaks.

They expect a simple interface, clear feedback, and easy game restarts without complications.

---

## 2. User Journeys

### User Journey – Office Worker

1. Opens the TicTacToe web application in a desktop browser.
2. Starts or joins a game session.
3. Waits for the second player to connect.
4. Makes a move by clicking on a field when it is his turn.
5. Communicates with the child via voice call during gameplay.
6. Continues playing until the game ends.
7. Sees the result (win, lose, or draw).
8. Restarts the game if desired.

---

### User Journey – Student

1. Opens the TicTacToe web application on a smartphone.
2. Joins the existing game session.
3. Waits for the opponent to make a move.
4. Makes a move by tapping on the game board.
5. Reacts to the opponent’s moves in real time.
6. Continues until the game ends.
7. Views the result (win, lose, or draw).
8. Restarts the game if both players agree.
